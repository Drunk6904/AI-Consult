## 1. 项目概述

### 1.1 项目简介
AI智能客服系统，目标是提供基于 RAG（检索增强生成）的企业问答服务，支持用户鉴权、会话管理、消息持久化和多渠道扩展。

### 1.2 技术栈
- 后端：Spring Boot 4.0.3、Java 21、Spring Security、JWT、Spring Data JPA
- 前端：Vue 3、Vite、TypeScript、Element Plus
- 数据库：PostgreSQL
- AI 调用：当前为 Mock 模式，预留真实 RAG 接入能力

### 1.3 默认端口
- 后端：`http://localhost:8080`
- 前端：`http://localhost:5173`
- PostgreSQL：`localhost:5432`
- AI 服务预留：`http://localhost:8081`

## 3.7 可能需要补充的接口说明（建议）

说明：以下接口用于支撑“未实现功能”的落地，属于建议清单，可按实际进度裁剪。

### A. 知识库与文档管理
1. 新建知识库
- `POST /api/knowledge/bases`
- 用途：创建 `public/private` 知识库
- 请求体示例：`{ "name": "公开知识库", "type": "public", "description": "面向游客" }`
- 返回建议：`id/name/type/createdAt`

2. 查询知识库列表
- `GET /api/knowledge/bases`
- 用途：前端选择检索范围，后台查看可用知识库
- 返回建议：支持按 `type` 过滤

3. 上传文档
- `POST /api/knowledge/documents/upload`（`multipart/form-data`）
- 参数：`file`、`knowledgeBaseId`
- 用途：文档入库并触发解析
- 返回建议：`documentId/status/fileType`

4. 重建索引/向量化
- `POST /api/knowledge/documents/{documentId}/index`
- 用途：手动触发分块、向量化、写入检索索引
- 返回建议：`chunkCount/vectorDim/status`

### B. RAG 问答
1. 统一问答接口（非流式）
- `POST /api/ask`
- 请求体建议：`conversationId`、`query`、`isRegistered`
- 返回体建议：`answer`、`confidence`、`sources[]`、`latencyMs`

2. 流式问答接口（可选）
- `POST /api/ask/stream`
- 用途：前端逐字展示回答，改善体验
- 协议建议：SSE

3. 检索调试接口（内部）
- `POST /api/rag/retrieve`
- 用途：仅返回召回片段与分数，便于调优 Top-K 和阈值
- 返回建议：`chunks[] {docName,snippet,score,locator}`

### C. 问答日志与反馈
1. 问答日志查询（管理端）
- `GET /api/question-logs`
- 用途：查看问题、回答、置信度、来源、耗时
- 查询参数建议：`from/to/userId/minConfidence/page/size`

2. 用户反馈接口
- `POST /api/question-logs/{id}/feedback`
- 用途：收集“有帮助/无帮助”与纠错文本，反哺知识库
- 请求体示例：`{ "helpful": false, "comment": "回答不准确" }`

### D. 统计分析
1. 汇总统计
- `GET /api/analytics/summary`
- 建议返回：`totalQuestions`、`todayQuestions`、`registeredQuestions`

2. 热门问题
- `GET /api/analytics/top-questions`
- 建议返回：`[{query,count}]`

3. 低置信度占比
- `GET /api/analytics/low-confidence`
- 建议返回：`lowConfidenceCount`、`totalCount`、`ratio`

### E. 运行保障（建议）
1. 数据库健康检查
- `GET /api/health/db`
- 用途：快速判断 PostgreSQL 可用性

2. AI 依赖健康检查
- `GET /api/health/ai`
- 用途：验证外部 AI 服务连通性与超时状态

3. 配置自检（开发环境）
- `GET /api/health/config`
- 用途：检查 `ai.mock`、`datasource`、`cors` 等关键配置是否生效

### 接口约定建议
- 统一响应结构：`code/message/data/timestamp`
- 分页参数统一：`page/size/sort`
- 管理接口默认鉴权（JWT）
- 检索和问答接口记录 `traceId`，便于定位问题

## 5. 关键配置说明

```properties
spring.datasource.url=jdbc:postgresql://localhost:5432/ai_consult
spring.datasource.username=postgres
spring.datasource.password=postgres
spring.jpa.hibernate.ddl-auto=update
spring.jpa.open-in-view=false

jwt.secret=ReplaceWithAtLeast32CharactersSecretForJwtDemo123456
jwt.expireMinutes=1440

ai.base-url=http://localhost:8081
ai.mock=true

app.cors.allowed-origins=http://localhost:5173
```

说明：
- `ai.mock=true`：返回模拟数据，用于前后端联调
- `ai.mock=false`：启用真实 AI 接口（需要先完成 RAG/AI 服务）

## 6. 注意事项

### 6.1 常见启动失败原因
1. 数据库未启动
- 典型报错：`Connection to localhost:5432 refused`
- 处理：启动 PostgreSQL，并确认 `TcpTestSucceeded : True`

2. Java 版本不匹配
- Spring Boot 4.x 需要 Java 21
- 用 `mvn -v` 检查 Maven 实际使用的 JDK

3. Maven 仓库镜像超时
- 若镜像频繁 504，可切到 Maven Central 直连

### 6.2 联调建议
- 先在 Mock 模式跑通注册/登录/会话/聊天全链路
- 再切换真实 AI（`ai.mock=false`）进行 RAG 联调
- 每次改动数据库配置后，建议重启后端
