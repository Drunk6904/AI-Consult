## 第2页：项目背景与目标

**标题：** 为什么我们需要AI智能客服？

**要点：**

| 痛点维度 | 现状数据 | 业务影响 |
|---------|---------|---------|
| **时间黑洞** | 机械劳动占比58%（4.7小时/天） | 顾问疲于应付，无精力做高价值工作 |
| **工具樊篱** | 5+系统切换，日均切换50+次 | 每次切换30-60秒恢复上下文 |
| **知识断层** | 70%复杂问题需跨部门确认 | 等待响应5-15分钟，客户流失率+30% |
| **责任焦虑** | 报价/政策解读出错=个人背锅 | "AI要是报错了价，这锅我背得起吗？" |

**核心目标：**
- 🎯 报价响应从4.5分钟压缩至15秒
- 🎯 80%重复问题AI自动托管
- 🎯 准确率100%，出错可追溯到系统

## 第3页：方案概览

**标题：** AI智能客服——闭环服务流程

**要点：**

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  知识库搭建  │────▶│  多渠道问答  │────▶│  用户权限   │
│ (PDF/Excel) │     │ (Web/微信)  │     │ (注册/未注册)│
└─────────────┘     └─────────────┘     └──────┬──────┘
                                                │
┌─────────────┐     ┌─────────────┐            │
│  数据洞察    │◀────│  咨询记录   │◀───────────┘
│ (TOP10/统计)│     │  日志记录   │
└─────────────┘     └─────────────┘
```

**四大核心模块：**
1. **知识库搭建**：PDF/Excel/Word一键上传，自动解析+向量化
2. **多渠道问答**：Web悬浮窗（已上线）+ 微信公众号（规划中）
3. **用户权限区分**：未注册用户仅访问30%公开知识，注册用户解锁完整服务
4. **咨询记录分析**：自动统计TOP10问题、响应时长、转化率

## 第5页：核心功能演示点（二）

**标题：** 功能演示：权限分级 + 人机协作

**要点：**

| 用户类型 | 访问权限 | 典型场景 |
|---------|---------|---------|
| **未注册用户** | 仅访问30%公开知识（产品介绍、基础价格） | 浏览官网，初步了解产品 |
| **注册用户** | 访问完整知识库 + 历史订单关联 | 老客户升级咨询，AI知道"他买过啥" |
| **VIP客户** | 优先转资深顾问 + 个性化推荐 | 高价值客户，享受专属服务 |

**人机协作机制：**
- 置信度<95% → 自动转人工
- 关键词触发（"折扣"、"法务"）→ 强制转人工
- 转人工时打包：客户画像 + 对话历史 + AI查询结果

## 第7页：安全与合规规划

**标题：** 安全与合规：让用户"敢用、愿用"

**要点：**

| 安全维度 | 具体措施 | 实施状态 |
|---------|---------|---------|
| **账号安全** | JWT无状态认证 + 密码哈希存储 | ✅ 已实现 |
| **权限控制** | 未登录用户访问限制（<30%） | ✅ 已实现 |
| **数据安全** | 客户敏感信息脱敏（手机号掩码） | 📋 规划中 |
| **操作审计** | 所有查询操作留痕（谁、何时、查什么） | ⏳ 待完成 |
| **知识保密** | 内部政策文档防泄漏（禁止复制/下载） | 📋 规划中 |
| **内容风控** | 高风险问题强制转人工（价格/合同） | ⏳ 待完成 |

**核心原则：**
> "宁可多转人工，不可漏转" —— 置信度<95%即转人工，确保100%准确
